# 🎯 电视知识竞赛抢答系统

**主持人控制大屏，选手用手机抢答** — 支持实时抢答、计时、计分，适合电视节目、课堂互动、团建活动。

> 当前版本：**v0.5.2** | GitHub：[ruccsf/quiz-buzzer](https://github.com/ruccsf/quiz-buzzer) | 更新：2026-05-31

---

## 📦 快速开始（3 步）

### 1. 安装 Node.js

> **已有 Node.js 的跳过此步。**

从官网下载安装：[https://nodejs.org](https://nodejs.org)（选择 LTS 版本）

安装完成后打开终端（cmd / PowerShell）验证：

```bash
node --version
npm --version
```

两条命令都显示版本号即表示成功。

### 2. 下载项目

```bash
# 方式一：从 GitHub 克隆
git clone https://github.com/ruccsf/quiz-buzzer.git
cd quiz-buzzer

# 方式二：直接下载 ZIP
# 打开 https://github.com/ruccsf/quiz-buzzer → Code → Download ZIP → 解压
```

### 3. 安装依赖并启动

```bash
# 安装依赖（只需执行一次）
npm install

# 启动服务
npm start
```

终端显示以下内容即启动成功：

```
🎯 抢答系统已启动！
   本地访问: http://localhost:3000
   局域网访问: http://<本机IP>:3000
   数据文件: rooms-data.json
   按 Ctrl+C 停止
```

> **Windows 用户**也可以直接双击 `start.bat` 一键启动。

## 🚀 使用指南

### 主持人 — 创建房间

1. 打开 **http://<电脑IP>:3000** → 点击 **🏆 主持人**
2. 输入比赛名称（如"谁是学霸"），点击 **创建房间**
3. 自动进入大屏仪表盘，右上角显示 **4 位房间号**（点击 📋 一键复制）
4. 告诉选手房间号，等待他们加入

### 选手 — 加入抢答

1. 手机上打开 **http://<电脑IP>:3000** → 点击 **📱 选手加入**
2. 输入房间号 + 昵称，点击 **加入房间**（或扫描大屏二维码，房间号自动填好）
3. 等待主持人开始抢答，出现红色按钮后点击即可

### 大屏展示 — 电视投屏

1. 电脑/电视打开 **http://<电脑IP>:3000** → 点击 **📺 大屏展示**
2. 输入房间号，点击 **连接大屏**（📥 粘贴按钮 + 📋 复制按钮）
3. 全屏显示比赛实况：实时排名、抢答状态、分数变化
4. 适合投影到电视或大屏幕供观众观看

### 游戏流程

```
主持人出题（输入题目+答案 → 发布）→ 点击"开始抢答" → 倒计时开始
                                                      ↓
                                         选手手机点击红色按钮抢答
                                                      ↓
                                 主持人判定 → ✅ 正确(+10分) / ❌ 错误(-5分)
                                                      ↓
                                             答案自动公布给所有人
                                                      ↓
                                               ⏭️ 下一题 → 循环
```

### 主持人控制面板

| 功能 | 按钮 | 说明 |
|------|------|------|
| 发布题目 | 📝 题目输入 + 📤 发布题目 | 输入题目文本和正确答案，发布到大屏和选手端 |
| 题库选取 | 📚 题库 | 从题库中选取题目，自动填入题目和答案（需管理密码验证） |
| 开始抢答 | 🚀 开始抢答 | 启动倒计时，选手可抢答 |
| 倒计时预设 | 5s / 10s / 15s / 20s / 30s | 点击切换倒计时时长 |
| 判对 | ✅ 正确 (+10) | 抢答者回答正确，加 10 分，同时公布答案 |
| 判错 | ❌ 错误 (-5) | 抢答者回答错误，减 5 分，同时公布答案 |
| 下一题 | ⏭️ 下一题 | 重置状态，清空题目，开始新一题 |
| 手动调分 | +5 / +10 / −5 | 直接调整任意选手分数 |
| 移除选手 | ✕ | 将选手从房间移除 |
| 二维码控制 | 📱 显示/隐藏二维码 | 大屏展示二维码，选手扫码加入 |

### 📚 题库管理（v0.4 新增）

主持人可以预先录入题目和答案，比赛中直接选取发布：

1. 首页点击 **📚 题库管理** → 输入房间号 + 管理密码（在 `config.json` 中配置）
2. **添加题目**：逐题输入题目+答案，或 **CSV 批量导入**
3. **比赛中使用**：主持人大屏点击「📚 题库」→ 输入管理密码 → 点击题目自动填充，一键发布
4. 已发布过的题目会标记 **✅**，防止重复出题（但仍可再次选用）

### 📋 一键复制与粘贴（v0.4 新增）

所有显示房间号的位置都配有 **📋 复制按钮**：
- host 页面顶部房间号 / 已有房间列表
- 选手页房间号 / 大屏页房间号 / 题库页房间号

所有需要输入房间号的位置都配有 **📥 粘贴按钮**：
- 选手加入页 / 大屏连接页 / 题库登录页

### 👤 选手头像上传（v0.5 新增）

选手可以上传或选择预设头像，实时显示在 host 和大屏上：

1. 选手页点击头像区域 → 弹出 **头像选择器**
2. **预设头像**：10 个都市白领形象（5 男 5 女），西装衬衫 + 干练发型
3. **上传照片**：浏览器 Canvas 自动压缩（200px, quality 0.5），手机原图 → ~30KB
4. 上传后实时广播 → host 大屏卡片、分数列表、排名、选手本人页全部更新
5. **删除房间时自动清理头像文件**

### 📺 大屏排行榜（v0.5 重做）

大屏展示页全面升级：

- **统一排行榜**：选手卡片和排名合并为 TV 风格列表，按分数自动排序
- **阶梯缩放**：排名越高行越宽、头像越大（第 1 名 72px → 第 8 名 44px）
- **🥇🥈🥉 发光徽章**：前三名金/银/铜色渐变色徽章带阴影发光
- **👑 皇冠动画**：第 1 名头顶悬浮皇冠，弹跳摇摆动画
- **同分同排名**：同分者并列名次，后面跳号（如 1, 1, 3）
- **分数变化提示**：+10 / -5 标签 3 秒后自动消失
- **FLIP 动画**：排名变化时行平滑移动过渡

### 📱 二维码扫码加入（v0.5.2 新增）

选手无需手动输入房间号，扫码即可加入：

1. **自动 IP 检测**：服务端启动时自动获取本机局域网 IP（优先 `192.168.x.x`），排除虚拟网卡和 VPN
2. **主持人控制**：host 控制面板点击「📱 显示二维码」→ 大屏全屏显示二维码
3. **大屏二维码**：260x260 大尺寸、深色背景、房号提示，选手举起手机一扫直达
4. **选手直达**：扫码后 `player` 页面自动填充房间号，只需输入昵称即可加入
5. **隐私保护**：二维码默认隐藏，主持人可控显示/隐藏，不干扰比赛

### 🔧 版本号统一管理 & 密码安全（v0.5.1 新增）

- **版本号统一管理**：版本号写入 `config.json` 的 `version` 字段，`index.html` 使用 `{{VERSION}}` 占位符，服务端启动时自动注入，一处修改全局生效
- **密码 MD5 存储**：管理密码不再明文存储，`config.json` 中保存 MD5 哈希值，校验时先哈希再比较
- **默认密码**：`96198`（MD5：`bc133f0a895e4cfdf51008b3de167a9e`）

### 大屏展示特色

- **题目展示**：28px 大号字体，题目和答案分两行显示
- **实时排名**：全选手排行榜 🥇🥈🥉 动态发光
- **答对撒花**：选手回答正确时，屏幕飘落 60 个彩色粒子 🎉
- **答案公布**：判分后答案自动在题目下方以绿色大字显示

## 🌐 局域网多设备使用

1. 电脑启动服务
2. 查电脑的局域网 IP：

```bash
# Windows
ipconfig | findstr "IPv4"

# Mac / Linux
ipconfig getifaddr en0   # Mac
hostname -I              # Linux
```

3. **访问首页**：所有设备打开 `http://<电脑IP>:3000`（手机/电脑/电视均可）

4. **角色分流**：
   - 🏆 **主持人**：点击首页的"主持人"卡片 → `/host`
   - 📱 **选手**：点击首页的"选手加入"卡片 → `/player`
   - 📺 **大屏展示**：点击首页的"大屏展示"卡片 → `/screen`

> 示例：电脑 IP 为 `192.168.50.145`，则选手访问 `http://192.168.50.145:3000`

## 💾 数据持久化

所有房间数据自动保存到项目目录下的 `rooms-data.json` 文件中，包括：

- 房间号和比赛名称
- 选手名称、分数、颜色
- 已完成的轮次和抢答历史

**服务器重启后选手可重新接入考场**：

1. 再次启动服务（数据自动恢复）
2. 主持人打开 `/host`，点击"已有房间"列表中对应的房间
3. 选手打开选手页面，输入房间号 + 原昵称，自动恢复身份和分数

## ⚙️ 高级配置

### 修改管理密码

管理密码存储在 `config.json` 中，使用 **MD5 哈希** 保存（非明文）：

```bash
# 生成 "你的密码" 的 MD5 值
node -e "const c=require('crypto');console.log(c.createHash('md5').update('你的密码').digest('hex'))"
```

将输出的哈希值替换 `config.json` 中的 `adminPassword` 字段即可。

### 修改端口号

```bash
# 启动时指定端口
PORT=8080 npm start

# Windows PowerShell
$env:PORT=8080; npm start
```

### 内网穿透（外网访问）

使用 [ngrok](https://ngrok.com) 或 frp 将本地服务暴露到外网：

```bash
ngrok http 3000
```

生成的外网 URL 发给选手即可远程加入。

## ⚡ 常见问题

**Q：Chrome 浏览器报 socket.io 400 错误无法连接？**
> 系统使用 WebSocket 直连，不支持 HTTP 长轮询降级。请使用 Chrome 最新版本，检查是否有扩展拦截 WebSocket。

**Q：选手页面显示"连接中..."不动？**
> 确认手机和电脑在 **同一 Wi-Fi 网络**，并且访问的是电脑的局域网 IP（不是 localhost）。选手访问首页后点"选手加入"进入加入页。

**Q：主持人页面创建房间没反应？**
> 打开浏览器开发者工具（F12）→ Console，看是否有 JS 报错。检查 `node server.js` 是否正常运行。

**Q：选手加入后 host 不显示？**
> 选手的页面跳转（首页→选手页）会自动触发重连，可能延迟 1-2 秒。刷新选手页面即可。

**Q：服务器重启后房间还在吗？**
> 房间数据保存在 `rooms-data.json`，重启后自动加载。主持人去 `/host` 查看"已有房间"列表即可重新进入。

**Q：最多支持多少选手？**
> 最多 **8 人**同时加入一个房间，可根据需要调整服务端代码中的限制。
